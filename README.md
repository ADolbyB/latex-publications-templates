## ðŸ› ï¸ Environment Setup

Choose your LaTeX workflow based on your preferences and needs.

### ðŸ’» Local Development (Recommended for Power Users)

**Option 1: VS Code on Linux Mint**

1. **Install VS Code** - [Download VS Code](https://code.visualstudio.com/)
   - Lightweight and fast
   - Excellent LaTeX support

2. **Add LaTeX Workshop Extension**
   - [LaTeX Workshop](https://marketplace.visualstudio.com/items?itemName=James-Yu.latex-workshop)
   - Auto-compilation on save
   - PDF preview in editor
   - Syntax highlighting and IntelliSense

3. Add all LaTeX packages to your installation: 
```bash
sudo apt-get install texlive-full
```

**Why this setup?**
- âœ… Fastest compilation
- âœ… Full control over packages
- âœ… Works offline
- âœ… Git-friendly workflow

**Option 2: TeXMaker (Standalone Editor)**

- **Install** - [Download TeXMaker](https://www.xm1math.net/texmaker/)
- Dedicated LaTeX IDE
- Built-in PDF viewer
- Still requires `texlive-full` for PDF compilation

> ðŸ’¡ **Important:** `texlive-full` includes all dependency packages, this minimizes PDF compilation errors.

## ðŸ§  Practical LaTeX Tips  

Even experienced LaTeX users forget these fundamentals:

| Tip | Why It Matters |
|-----|----------------|
| âœ… **Use `.bib` files** | Clean, scalable citations that auto-format |
| âœ… **Split large documents** | Multiple `.tex` files improve readability |
| âœ… **Prefer vector graphics** | PDFs and SVGs scale perfectly |
| âœ… **Use version control** | Git + LaTeX = perfect collaboration |
| âœ… **Comment your code** | Future you will thank present you |
| âœ… **Compile multiple times** | References need 2-3 passes to resolve |
| âœ… **Keep packages minimal** | Only load what you actually use |
| âœ… **Use semantic commands** | `\emph{}` not `\textit{}` for meaning |

These small habits make a **massive difference** in long-term productivity.
